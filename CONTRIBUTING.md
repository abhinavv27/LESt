# 🤝 Contributing to Lex-Contrast

Thank you for your interest in the Lex-Contrast Multi-Agent Swarm project! As we are working with an autonomous swarm architecture, contribution protocols differ from standard open-source logic.

---

## 🛰️ How to Contribute

### 1. Swarm Development
- We are actively looking for specialists to add to our agent kernels (e.g., specific agents for Tax Law, Privacy, or Employment Law).
- New agents should follow the metadata interface defined in `useAgentStream.ts`.

### 2. UI/UX Evolution
- Innovations in cyber-neumorphic design tokens and 3D visualization using Three.js are highly encouraged.
- Ensure all new components use the established `cyber-neo` CSS variables in `globals.css`.

### 3. Protocol Testing
- Fork the repository, run `npm run dev`, and upload custom contracts (`doc_a.txt`, `doc_b.txt`) to stress-test the `Cortex Supervisor`'s conflict detection logic.

---

## 🛠️ Local Development

1. Fork the repo.
2. Install dependencies: `npm install`.
3. Create a feature branch: `git checkout -b feature/new-specialist-agent`.
4. Run locally: `npm run dev`.
5. Submit a pull request with high-fidelity screenshots of the new agent appearing in the SwarmHUD.

## 📜 Code of Conduct
Lex-Contrast maintains an open, professional environment for distributed intelligence development. Respect all agents and human collaborators.
